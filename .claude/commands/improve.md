# /improve — Melhorar funcionalidade existente

## Processo

### 1. Entender o estado atual

Leia o componente/arquivo antes de propor qualquer mudança.

### 2. Propor melhoria

```
💡 MELHORIA PROPOSTA

ATUAL: [o que existe hoje]
PROPOSTA: [o que vai melhorar]
ARQUIVOS: [src/...]
IMPACTO: [visual / performance / UX / código]
RISCO: [baixo/médio — e por quê]
```

Aguarde aprovação.

### 3. Implementar

- Alterar apenas o que foi aprovado
- Não aproveitar para refatorar código não relacionado
- Manter TypeScript tipado

### 4. Confirmar

```
✅ MELHORIA APLICADA

ARQUIVO: [src/...]
MUDANÇA: [descrição]
COMO VER: [onde testar no app]
```
